Redis持久化方式——AOF与RDB使用总结

2018-06-08

Redis所有数据保持在内存中，对内存中的数据异步保持到磁盘上的过程叫做持久化。常见的持久化方式包括两种： 快照就是把某个时间节点的数据进行备份；写日志就把每次对数据库的操作都进行备份。RDB和AOF是Redis持久化的两种方式。

<!--more-->

1 RDB工作原理

触发Redis进行RDB备份的方式有三种，分别是：

  1. save命令（同步）
  2. bgsave（异步）
  3. 自动

1.1 save命令方式过程

save是同步命令，其他客户端必须等待save命令执行完。如果已经存在老的RDB文件，则替换成新生成的。

1.2 bgsave命令（异步）过程

bgsave是异步命令，fork（Linux）命令会开启一个守护进程（其实fork是一个阻塞命令，当然fork很快），在执行bgsave时，Redis服务器依然可以响应客户端。

2 AOF工作原理

AOF属于写快照的方式，将每次对Redis的写操作都记录在日志文件中，当需要恢复数据再根据AOF文件进行恢复。大致过程如下：

2.1 AOF的三种策略

  1. always # 只要进行了写操作，立马写入AOF文件
  2. everysec # 每秒将缓冲区的写操作写入AOF文件
  3. no # 由操作系统决定什么时候将缓冲区的数据写入AOP文件
