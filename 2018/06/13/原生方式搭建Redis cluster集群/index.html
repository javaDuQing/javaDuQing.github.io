【24】原生方式搭建 Redis cluster 集群

2018-06-13

搭建Redis cluster集群方式两种：

  1. 原生命令安装
  2. 官方工具安装

推荐使用官方工具搭建，我这是为了了解其中的原理，体会过程，所以就使用第一种方式搭建了

搭建的过程包括：

  1. 配置开启节点
  2. meet操作
  3. 指派槽
  4. 配置主从

2 meet操作

cluster meet命令被用来连接不同的开启集群支持的Redis节点。基本的思想是每个节点默认都是相互不信任的，
可以通过meet操作，让一个节点会面另一个节点，让该集群中的节点都相互知道。

该过程使用gossip协议，只要一个节点meet了其他所有节点。则集群中的所有节点都相互了解了。
如下图，A节点分别meet B，met C节点。然后A B C节点之间就都互通了。（很神奇，对gossip协议不是很了解，具体细节没深入研究）

通过cluster slots可以查看master节点，以及master负责的槽，以及master节点的slave节点。可以看出 第一个master负责的槽的从0-5461，slave节点是7003。

在Redis cluster集群中，一般slave节点只是启动故障转移的作用。不能进行读写操作。如果想要有读操作，需要执行readonly命令。不建议使用cluster的读写分离的作用。
