Tomcat 集群下，使用 Redis 解决单点登录问题

2018-06-20

单个tomcat项目登录流程如下：

过程可总结如下：

  1. 用户填写用户名、密码，点击登录按钮
  2. 后端根据用户填写的信息，到数据库中查找该用户是否存在，密码是否正确
  3. 如果用户存在且密码正确：则将用户信息存储到session中
  4. 否则：提示用户用户名或密码错误

对于单个tomcat项目通过session的方式判断用户是否登录的逻辑如下：

  1. 浏览器第再次访问服务器时，携带session ID
  2. 服务器从请求中提取出Session ID，判断该session ID是否保存了用户信息，如果有，则已经登录；如果没有，则未登录

tomcat集群，单点登录问题

如果是tomcat集群，使用Nginx做反向代理和负载均衡。那么我们的请求会打在不同的tomcat上（根据Nginx负载均衡策略），如果用户在一个tomcat登录，第二次请求打到了另外一台tomcat，两个tomcat产生的session ID是不一样的，该怎么办呢？如何做到只需要在一台tomcat上登录？

Nginx负载均衡策略——IP Hash可以解决，本文不使用这种解决方案

cookie + Redis 解决单点登录

下面来看使用的cookie+Redis解决单点登录的时序图，与前一幅图主要有两点区别：

  1. 登录成功的话，将session ID和用户信息存入Redis中
  2. 登录成功的话，写一个cookie给浏览器

判断用户是否登录的详细过程如下：

  1. 浏览器再次请求tomcat（可能是集群中的任一个），携带cookie信息
  2. 服务器根据cookie name寻找cookie是否存在
  3. 如果存在，则根据token（session ID），从Redis中取出用户信息
  4. 如果不存在，则用户未登录
