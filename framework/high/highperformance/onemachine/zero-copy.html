    零拷贝技术

    奇风 2023/2/7

    # 1 传统数据读写与网络发送的瓶颈

    如下伪代码会进行几次拷贝： File.read(file, buf, len); Socket.send(file, buf, len); 会进行4次拷贝 会进行几次上下文切换呢？（内核态与用户态之间的切换），也是4次：

    1. File.read这个代码是在用户态先执行的，然后要切到内核态进行第一次拷贝
    2. 把从内核态拷贝的数据 放到 应用程序的buf中，这是第二个切换
    3. Socket.send是在用户态执行的，要从用户态把buf中的数据拷贝到内核态的套接字发送缓冲区
    4. 用户态把buf中的数据拷贝到内核态的状态要返回给用户态

    # 2 拷贝的优化

    讲道理2，3两步拷贝不需要，可以优化掉 零拷贝（Zero-copy）：是指计算机执行操作时，CPU不需要先将数据从某处内存复制到另外一个特定区域；主要是减少不必要的数据拷贝、和不必要的上下文切换

    # 2.1 Linux——MMAP内存映射

    3次拷贝，4次上下文切换，Java代码如下：

    public static String path = "F:\\mmap"; // 需要进行内存映射的文件目录
public static void main(String[] args) throws IOException {
    File file1 = new File(path, "1");
    RandomAccessFile accessFile = new RandomAccessFile(file1, "rw");
    int len = 2048;
    MappedByteBuffer mmap = accessFile.getChannel()
        .map(FileChannel.MapMode.READ_WRITE, 0, len);
    mmap.put("k".getBytes());
    mmap.put("i".getBytes());
    mmap.flip();
}

    mmap是一种内存映射文件的方法，即将一个文件或者其它对象映射到进程的地址空间，实现文件磁盘地址和进程虚拟地址空间中一段虚拟地址的一一对映关系。实现这样的映射关系后，进程就可以采用指针的方式读写操作这一段内存，而系统会自动回写脏页面到对应的文件磁盘上，即完成了对文件的操作而不必再调用read,write等系统调用函数。相反，内核空间对这段区域的修改也直接反映用户空间，从而可以实现不同进程间的文件共享

    # 2.2 Linux——sendfile

    3/2次拷贝：3次拷贝如图所示；Linux 2.4支持两次拷贝，文件读取缓冲区直接把地址发送给网卡（第一次使用 DMA 引擎从文件拷贝到内核缓冲区，第二次从内核缓冲区将数据拷贝到网络协议栈；内核缓存区只会拷贝一些 offset 和 length 信息到 SocketBuffer，基本无消耗） （疑问：拷贝offset和length到socket buffer有啥用？） 2次上下文切换：用户态通知系统态进行文件发送，系统态通知用户态发送结果

    public static void main(String[] args) throws IOException {
    SocketChannel socketChannel = SocketChannel.open();
    socketChannel.connect(new InetSocketAddress("localhost", 8089));
    socketChannel.configureBlocking(true);

    String fileName = "C://Users//king.jpg";
    FileChannel fileChannel = new FileInputStream(fileName).getChannel();
    long transferCount = fileChannel.transferTo(0, 
                                fileChannel.size(), socketChannel);
    fileChannel.close();
    socketChannel.close();
}

    # 2.3 两种拷贝之间的区别

    1. mmap主要优化的是【 硬盘到应用之间的数据拷贝】
    2. sendfile优化的是 【硬盘到网卡之间的数据拷贝】
    3. mmap 适合小数据量读写，sendFile 适合大文件传输

    # 3 消息中间件的选择

    RocketMQ使用mmap是因为RocketMQ实现的很多功能要内存处理，例如消息过滤，顺序消息等 Kafka应该没在内存中做一些事情
